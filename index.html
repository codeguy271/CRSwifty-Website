CRSwifty

CRSwifty is a exploit that will be rolled out in RMA Shims. We will also provide binaries here, but currently it allows for modification of GBB Flags in readonly firmware, WITHOUT removing the write-protect screw.

Before running any CRSwifty commands, make sure to sudo initctl stop trunksd

Offline Instructions

  1. Download the latest build
  2. get my ec.ro.elf
  3. execute crswifty -f <random file> --sysinfo (it won't actually ping any servers or validate if the random file is a real firmware file) (save this, you WILL need it later)
  4. put the system info in the client
  5. download the signed image by pressing sign
  6. put the signed image in the shim
  7. Run crswifty -f signed.bin --offline --verbose

Actually disabling wp:

  1. Download the latest ec.RW.elf
  2. Go to the client
  3. Input RW as the type, and the sysinfo you have gotten before and download the signed image
  4. Put the signed image in the RMA Shim
  5. crswifty -f signed.bin -t rw --offline --verbose

How to build your own cr50 image for uploading

Create a chromiumOS chroot at https://chromium.googlesource.com/chromiumos/docs/+/master/developer_guide.md. After doing so follow these steps Run this from where you cloned the chromiumos source code.
  1. Modify the code in src/platform/cr50 to whatever you want.
  2. Enter the chroot by running cros_sdk
  3. Run cd ~/trunk/src/platform/cr50/
  4. Run make BOARD=cr50
  5. Exit the chroot
  6. From the directory you entered the chroot, run cp src/platform/cr50/build/cr50/RO/ec.RO.elf and use that as the firmware file input for crswifty.
